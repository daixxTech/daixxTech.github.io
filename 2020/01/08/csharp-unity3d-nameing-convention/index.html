C# Unity3D 命名规范（个人）

发表于 2020-01-08 分类于 编程规范

本文主要记录 C# 和 Unity3D 中的一些命名规范问题，文章内容会根据个人的学习进度进行更新。

个人认为好的编程习惯是非常重要的，整洁规范的代码可以显著地提高可读性，一方面方便开发人员对代码进行维护，另一方面阅读这样的代码可以使人身心愉悦。

PS：该命名规范有一定程度的个人主观因素，仅供参考。

通用

bool类型

一般根据含义以 is / can / has 等作为前缀。

代码块分区、说明

用于对代码块进行分区、说明。

简写

全称 简写

Transform Trans
Button Btn

Current Cur
Object Obj

类

使用帕斯卡命名规则。

类名应该是能够反映类功能的名词，或名词短语。

接口

使用帕斯卡命名规则。

以大写字母 I 开头，接口名应该是动词。

字段

使用驼峰命名规则。

字段一定是私有的，用 private 修饰。

属性

使用帕斯卡命名规则。

属性一定是公有的，用 public 修饰。

方法

使用帕斯卡命名规则。

方法名应该是动词，或者动宾短语。

一般根据字段含义以 Is / Can / Has 等作为字段名前缀。

响应事件

以 On 开头（如 OnClose，OnClick），OnXXX 表示当 XXX 发生时执行的函数。

判断方法名是否加 On 的简单方法（不谨慎，仅供参考）：

如果方法执行的内容和函数名相同则不加 On，不相同则加 On。
