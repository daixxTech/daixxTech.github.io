C# 中 Hashtable 和 Dictionary 的区别

发表于 2020-03-01 更新于 2020-03-02 分类于 CSharp

本文主要记录 C# 中 Hashtable 和 Dictionary<K, V> 的区别。

存储的数据类型

Hashtable

Hashtable 不支持泛型，不是类型安全的。

Key-Value 均为 object 类型，所以 Hashtable 支持任何类型的 Key-Value。

但也因此，Hashtable 在存储 / 检索值类型的数据时，会发生装箱 / 拆箱的操作，会消耗一定的性能。

Hashtable 中的元素是 DictionaryEntry 类型。

Dictionary<K, V>

Dictionary<K, V> 支持泛型，是类型安全的。

Key-Value 的数据类型在声明时就要指定。

Dictionary<K, V> 中的元素是 KeyValuePair<TKey, TValue> 类型。

存储数据与读取数据的顺序一致性

Hashtable

Hashtable 的数据存储顺序是按一定的算法得出的，因此，在绝大多数情况下，存储数据与读取数据的顺序是不一致的。

Dictionary<K, V>

Dictionary<K, V> 在只存储不删除的情况下，存储数据与读取数据的顺序一致。

在删除 A 数据后，存储 B 数据，此时 B 数据会填补之前 A 数据的位置。

因此，经过删除和存储操作后，存储数据与读取数据的顺序会产生不一致。

以不存在的 Key 在数据结构中取 Value

Hashtable

Hashtable 会返回一个 null 值。

Dictionary<K, V>

Dictionary<K, V> 会抛出 System.Collections.Generic.KeyNotFoundException 类型的异常。

线程安全性

Hashtable

Hashtable 是线程安全的。

Dictionary<K, V>

Dictionary<K, V> 不是线程安全的。
