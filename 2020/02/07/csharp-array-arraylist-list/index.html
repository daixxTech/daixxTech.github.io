C# 中 数组，ArrayList 和 List 的区别

发表于 2020-02-07 分类于 CSharp

本文主要记录 C# 中 数组，ArrayList 和 List 的区别，主要目的是明确 ArrayList 的定义。

数组

数组，即 Array，但是 C# 中的 Array 类是抽象类，并不能被实例化。

数组的逻辑结构为线性结构。

数组的存储结构为顺序结构，即数组在内存中是连续的，因此数组的索引速度快。

数组的赋值和修改简单，但是插入和删除比较麻烦。

声明数组的时候必须指定数组的长度，长度过长会造成内存浪费，过短会造成内存溢出。

因此当不清楚数组的长度时，使用起来会很麻烦，于是 C# 提出了 ArrayList 来克服这些缺点。

ArrayList

动态数组，即 ArrayList，实现了 IList 接口。

ArrayList 的长度是按照其中存储的数据来动态扩充与收缩的。因此在声明 ArrayList 时不需要指定其长度。

ArrayList 中可以存储不同类型的数据，举个例子：

可以看出，ArrayList 中可以插入不同类型的数据，这是因为 ArrayList 会将所有插入其中的数据作为 object 类型来处理。

因此对 ArrayList 中的值类型数据进行存储或检索时，会发生装箱和拆箱的操作，这是很大的性能开销。

在使用 ArrayList 处理数据时，很可能出现类型不匹配，因此 ArrayList 不是类型安全的。

在日常使用中，如果只是为了使用动态的数组，就不要使用 ArrayList。

类型安全

一个对象在定义时就会被分配一个类型。

在使用该对象时，只允许执行其类型所允许的操作，那么该语言是类型安全的。

类型安全不应该与 静态/动态类型 或 强/弱类型 混淆。

List

链表，即 List，实现了 IList 接口。

List 泛型在声明时就限制了其存储内容的数据类型，因此就没有了 ArrayList 中的装箱和拆箱操作，并且是类型安全的。
